You’re right to flag that. The icons should **not** be green. That likely happened because the new coach-mode helpers used **emerald** accents and, somewhere on the Home tiles, an inherited `currentColor` (e.g., `text-emerald-*`, `fill-emerald-*`, or `stroke-emerald-*`) leaked onto the Lucide icons. Many Lucide icons use `stroke="currentColor"`, so any parent text color turns the icon that color.

Here’s a clean, surgical fix that restores your **orange→black gradient** icon aesthetic, prevents future leaks, and keeps the coach-mode flashes green **without** recoloring icons.

## 2) Lock icons to brand gradient (reusable wrapper)

Use a small wrapper so every tile icon renders **orange→black gradient** consistently, regardless of parent text color.

Create: `client/src/components/BrandTileIcon.tsx`

```tsx
import { ReactNode } from "react";

/**
 * Wrap Lucide icons to enforce orange→black gradient fill/appearance.
 * Works by using bg-clip-text; for stroke-only icons, we show the icon in white
 * over a small gradient disk for visual consistency across all icons.
 */
export default function BrandTileIcon({ children }: { children: ReactNode }) {
  return (
    <div className="relative inline-flex items-center justify-center">
      {/* Gradient disk behind the icon */}
      <span className="absolute inset-0 rounded-full bg-gradient-to-br from-[#FF7A00] to-black opacity-100 scale-110" />
      {/* Icon foreground kept white for clarity on gradient */}
      <span className="relative text-white">
        {children}
      </span>
    </div>
  );
}
```

Usage example in `DashboardNew.tsx`:

```tsx
import { Calculator, Activity, ScanBarcode } from "lucide-react";
import BrandTileIcon from "@/components/BrandTileIcon";

// ...
<button className="tile">
  <BrandTileIcon><Calculator className="w-6 h-6" /></BrandTileIcon>
  <span>Macro Calculator</span>
</button>

<button className="tile">
  <BrandTileIcon><Activity className="w-6 h-6" /></BrandTileIcon>
  <span>My Biometrics</span>
</button>

<button className="tile">
  <BrandTileIcon><ScanBarcode className="w-6 h-6" /></BrandTileIcon>
  <span>Scan Barcode</span>
</button>
```

Do the same in `BottomNav.tsx` so all 4 tabs match the brand look.

> Why this works: we never rely on `currentColor`. The icon stays **white**, the gradient sits behind it. No parent text color can turn it green.

## 3) Make flash effects color-safe (don’t recolor icons)

Keep coach-mode flashes **box-shadow only**, no `text-emerald-*` on containers that hold icons.

**Ensure your flash CSS looks like this (no color set):**

```css
@keyframes pulseGreen {
  0%, 100% { box-shadow: 0 0 0 0 rgba(16,185,129,0.65); }
  50% { box-shadow: 0 0 18px 10px rgba(16,185,129,0.25); }

.flash-green, .flash-green-strong {
  animation: pulseGreen 1.1s infinite;
  border-radius: 0.75rem;
  /* No text/fill/stroke colors here */
}
```

If any guided component added `text-emerald-*` to a parent, change that to a **border/outline** or **ring** utility instead (e.g., `ring-emerald-400`) so icon color doesn’t change.
