The Issues I Found:
1. Category Tabs Are Getting Cut Off
The category tabs container has overflow-x-auto but they're being squeezed. The issue is the parent dialog content has max-h-[85vh] which is creating vertical compression, and when combined with all the other sections (header, macro targeting, search input, ingredient grid, selected ingredients display, custom input, buttons, progress bar), the category tabs are getting pushed into a tight space.

2. Ingredient Grid Is Only One Screen High
This line is the problem:

<div className="overflow-y-auto max-h-[50vh] mb-3 min-h-0">
The max-h-[50vh] is limiting the ingredient viewing area to only 50% of the viewport height. With all the other UI elements above and below it (Beach Body mode banner, macro targeting controls, category tabs, search, selected ingredients, custom input, buttons), there's barely any room left.

Summary:
The modal is trying to fit too many UI sections into a max-h-[85vh] container, and the ingredient grid is explicitly limited to max-h-[50vh], which means it can only show a small portion of the available ingredients at once. The category buttons aren't "hidden" per se, but they're in a cramped horizontal scroll area because there's not enough vertical space in the overall layout.

The root cause is vertical space management - not enough room allocated to the main ingredient selection grid relative to all the auxiliary UI elements.
